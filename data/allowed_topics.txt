Hệ Thống Quản Lý Giải Đấu Rikkei Esports (Refactoring & Testing)  
1. Bối cảnh nghiệp vụ 


Tại ban tổ chức giải đấu Rikkei Esports, hệ thống phần mềm dùng để ghi nhận kết quả các trận đấu đang gặp vấn đề nghiêm trọng. Một cựu nhân viên đã để lại một file code duy nhất chứa vòng lặp while khổng lồ. Code vi phạm nghiêm trọng các tiêu chuẩn PEP 8, không có cơ chế bắt lỗi (Exception Handling) khiến phần mềm thường xuyên sập khi trọng tài gõ nhầm điểm số. Tệ hơn, khi có lỗi xảy ra, không ai biết nguyên nhân vì hệ thống không hề ghi log (Logging).

Nhiệm vụ của bạn là đập đi xây lại (Refactor) hệ thống này thành một dự án đạt chuẩn kỹ sư chuyên nghiệp: Code sạch, chia module, bẫy lỗi triệt để, ghi log mọi hoạt động và có Unit Test bảo vệ logic cốt lõi.

Yêu cầu kỹ thuật bắt buộc: Lập trình viên phải định nghĩa và sử dụng các hàm (Function) riêng biệt cho từng nghiệp vụ.

3. Edge Cases (Bẫy dữ liệu) 
Bẫy 1 — Lỗi logic ẩn (Cần Debugging): Trong code cũ, khi tỷ số là 0-0 nhưng trạng thái vẫn bị gắn là "Completed" do logic kiểm tra sai. Bạn cần thiết lập Breakpoint trên IDE (VSCode/PyCharm) để debug và đảm bảo trận đấu chỉ "Completed" khi ít nhất 1 đội có điểm > 0, hoặc do trọng tài chủ động xác nhận.

Bẫy 2 — Nhập sai lựa chọn menu: Nếu nhập ngoài phạm vi 1-5, in ra cảnh báo và ghi log WARNING "Invalid menu choice selected".

Bẫy 3 — Dữ liệu rỗng: Cần dùng try...except KeyError trong quá trình trích xuất dữ liệu từ dictionary để phòng hờ trường hợp API trả về thiếu key score_a hoặc score_b.

4. Yêu cầu 
Với vai trò kỹ sư phần mềm, bạn phải thực hiện đầy đủ 3 phần sau:

(1) Phân tích & Clean Code (Bắt buộc)

Đặt tên biến và hàm chuẩn PEP 8 (snake_case cho biến/hàm).
Viết Docstrings cho tất cả các hàm (Mô tả chức năng, tham số, kiểu trả về).
Cấu hình thư viện logging của Python để ghi log ra một file tên là tournament_app.log với định dạng: [Thời gian] - [Cấp độ Log] - [Tin nhắn].
(2) Triển khai Code (Refactoring & Exception)

Viết source code chính vào file main.py.
Ứng dụng triệt để try-except tại tất cả các điểm có tương tác nhập liệu từ người dùng.
(3) Viết Unit Test (Bảo vệ logic)

Tạo một file riêng tên là test_matches.py.
Import thư viện unittest hoặc pytest.
Viết ít nhất 3 test cases cho hàm phụ trợ determine_winner(match):
Test case 1: Đội A thắng (VD: score_a = 2, score_b = 0).
Test case 2: Trận đấu hòa (VD: score_a = 1, score_b = 1).
Test case 3: Trận đấu chưa diễn ra (status = "Pending").
